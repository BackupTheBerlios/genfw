Creating a Generator Application

  1. Go back to the tutorial1.orderman project and in the model folder create a new Generator Application named orderman.genapp (New Wizard | Sympedia Generation | Generator Application).

  2. Add an Input element to the GenApp with the path model/orderman.beans. The property label Full Path is actually misleading since input paths can also be relative to the project that contains the genapp file. This makes copying the genapp file to other application easier! To be able to refer to elements of the genlib file, load the beans.genlib resource into the GenApp editor. Now select ECORE as the ContentProvider and BeanRules as a RuleSet to use.

    Figure 6. Generator Application created

  3. Recall that in the BeanBaseRule we direcetd the output of the generation to the src-gen folder of the GenApp project. Don’t forget to add the src-gen source folder to the tutorial1.orderman project now!

  4. Save the genapp file and right-click on it in the Package Explorer. In the context menu select Sympedia Generation | Generate. That’s it. Look into the src-gen folder. GenFw has created a package with the PersonBeanBase class:

    package tutorial.orderman; 
 
public abstract class PersonBeanBase 
{ 
  protected int id; 
  protected String name; 
  protected double income; 
 
  public PersonBeanBase(int id, String name, double income) 
  { 
    this.id = id; 
    this.name = name; 
    this.income = income; 
  } 
}
